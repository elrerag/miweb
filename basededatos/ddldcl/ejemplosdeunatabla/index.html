    Ejemplos de una tabla.

    • Solamente se muestran las consultas, la conexión la pueden extraer de las guías de repaso o cualquier video parecido al del siguiente enlace: https://www.youtube.com/watch?v=Zgq9JABTF8k.
    • Se expondrán una serie de ejemplos, que irán incrementando su dificultad, intentando cubrir los aspectos más recurrentes en los que necesitemos realizar una consulta a la base de datos.
    • Oracle 11g NO es sensible a mayúsculas o minúsculas, por lo tanto, las consultas pueden contener texto formado por mayúsculas, minúsculas o una mezcla de ambas. Cualquiera sea el caso, no influye en el resultado obtenido.
    • Partiremos con un campo y luego lo incrementamos para aprovechar una mejor visualización de las ventajas al aplicar alguna técnica.

    CRUD un campo.

    Crear.

    • Se necesita almacenar la información de una cuenta de usuario de una red social.
    • La información a almacenar es el nombre de usuario.

    Solución

    CREATE TABLE cuenta (nombre_usuario VARCHAR2(200));

COMMIT;
    • Podemos confirmar la estructura desde sqlpluss.
    SQL> describe cuenta;
 Name                      Null?    Type
 ----------------------------------------- -------- ----------------------------
 NOMBRE_USUARIO                     VARCHAR2(200)

SQL>

    Ingresar

    • Una vez creada la tabla, podemos agregar datos.
    INSERT INTO cuenta (nombre_usuario) VALUES ('lherrera');
commit;
    • El primer paréntesis indica el orden, en el que entregaremos los datos, en el segundo paréntesis irán los datos entregados según su tipo.
    • Además se puede ingresar otro dato y esta vez no irán los paréntesis.
    INSERT INTO cuenta VALUES ('otrousuario');
commit;

    Consultar

    • Podemos saber los registros que posee una tabla.
    SELECT * FROM cuenta;

    Resultado.

    NOMBRE_USUARIO
--------------------------------------------------------------------------------
lherrera
otrousuario
    • No es conveniente traer toda la información utilizando el *, ya que no se puede obtener información detallada de lo que se está consultando. Por este motivo que cuando se realiza una consulta, especificamos cada campo que consultemos.
    • La siguiente consulta, obtiene los mismos resultados.
    SELECT nombre_usuario FROM cuenta;

    Resultado.

    NOMBRE_USUARIO
--------------------------------------------------------------------------------
lherrera
otrousuario

    Actualizar

    • Para actualizar información debemos indicar tres cosas:
      • La tabla que se actualizará.
      • El valor de cada campo a actualizar, indicando a qué campo pertenece cada valor.
      • Una condición de filtrado de datos para que actualice aquellos datos que realmente se va a actualizar.
        • Si no se especifica el filtro, se aplicará a toda la columna.
    • En el siguiente ejemplo cambiaremos el valor del segundo usuario.
    UPDATE cuenta SET nombre_usuario = 'jlopez' WHERE nombre_usuario = 'otrousuario';
COMMIT;

    Si consultamos los datos como lo hicimos anteriormente, esta vez el resultado será:

    NOMBRE_USUARIO
--------------------------------------------------------------------------------
lherrera
jlopez

    Eliminar

    • Podemos eliminar los datos de una tabla, indicando dos cosas:
      • El nombre de la tabla que se le aplicará la eliminación de registros.
      • La condición de filtrado de filas a eliminar.
        • Si no se especifica el filtro, se eliminan todos los registros de la tabla.
    • Esto elimina la fila completa, para operaciones específicas sobre un atributo en particular, se realiza con update, delete, elimina la fila completa.
    DELETE FROM cuenta WHERE nombre_usuario = 'lherrera';
COMMIT;
    • Si ejecutamos el select, ahora el resultado será el siguiente;
    NOMBRE_USUARIO
--------------------------------------------------------------------------------
jlopez

    Borrar una tabla

    • A diferencia del comando anterior, esta instrucción, elimina la tabla.
    DROP TABLE cuenta;
COMMIT;
    • Si ejecutamos un select, esta vez, no nos podrá mostrar el contenido de un elemento que no existe.
    ERROR at line 1:
ORA-00942: table or view does not exist

    CRUD cinco campos.

    Crear

    • Creamos una tabla que almacena la información de un empleado.

    • Los atributos del empleado serán:

      • código
      • nombre
      • departamento
      • salario
      • fecha de ingreso
      • código jefe

    • Se indica una clave primaria para poder optimizar la búsqueda de registros y para controlar que los datos ingresados en este campo no se repitan.

    Solución

    CREATE TABLE empleado 
  ( 
     codigo        NUMBER, 
     nombre        VARCHAR2(200), 
     departamento  VARCHAR2(200), 
     salario       NUMBER, 
     fecha_ingreso DATE, 
     codigo_jefe   NUMBER, 
     PRIMARY KEY (codigo) 
  );
COMMIT;

    • Observamos lo siguiente en el problema:
      • La tabla contiene información que pertenece a la propia tabla, es decir que se encuentra relacionada con ella misma.
      • Cuando esto ocurre, es normal requerir en una sola consulta (no anidada), la información del objeto en cuestión y de sus relaciones.
      • Cuando tengamos a dos o más tablas relacionadas, podemos obtener datos al instante, fusionando la información en varios conjuntos, mediante un filtro.
      • La instrucción inner join llena la información de varias tablas secuencialmente dependiendo de lo que queremos filtrar.
      • Los pasos a seguir en la siguiente consulta son:
        • Llenar la tabla EMPLEADO a con las mismas condiciones que hemos visto hasta ahora.
        • Agregamos una fusión de datos con una tabla relacionada, en este caso la misma tabla EMPLEADO b; pero con otro alias.
        • Indicamos que esta tabla se llenará, siempre y cuando el CODIGO_JEFE de la tabla de resultados (conteo agrupado) sea igual al CODIGO de la tabla de fusión.
        • Por lo tanto EMPLEADO b, se llenara solo en los casos que se cumpla a.CODIGO_JEFE = b.CODIGO. Hay que tomar cada inner join como una lista nueva con información, obtenida con la condición seguida de ON, (pueden continuar con and).
        • Dentro del inner join, lo que se filtre por fuera, es decir en el where apunta a lo que venga directamente a EMPLEADO a, es decir a lo que venga después del From. Lo que venga después del inner join corresponde al filtro de la tabla EMPLEADO b.
        • Con inner join el resultado final depende finalmente de la intersección de las listas llenas de datos (EMPLEADO a y EMPLEADO b). Ambas cumplen con todas las condiciones. Es decir que el resultado final será una intersección entre los datos INNER JOIN EMPLEADO b ON a.CODIGO_JEFE = b.CODIGO.
        • Si cuando se están llenando las dos listas con información, no se cumple con lo expresado en ON a.CODIGO_JEFE = b.CODIGO, no se llena ni a ni b, quitando la fila de información que no cumple pasando a la siguiente iteración.
        • Si se quiere que no se filtren los datos de la tabla 1, se realiza un left join, si no se desea filtrar los resultados de la segunda tabla aunque no existan coincidencias en el on, se realiza un right join, aunque lo usualmente utilizado es solamente inner join.
    SELECT
    b.CODIGO,
    b.NOMBRE,
    COUNT(a.codigo) cantidad_empleados
FROM
    EMPLEADO a
    INNER JOIN EMPLEADO b ON a.CODIGO_JEFE = b.CODIGO 
GROUP BY b.CODIGO, b.NOMBRE;
