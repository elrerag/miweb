    Manejo de sesiones

    Competencias:

    • Entender diferencia entre cookie y sesión.
    • Aplicación de variables de sesión en entorno de prueba.

    Introducción

    El protocolo http técnicamente no mantiene el estado en la aplicación, esto quiere decir que olvidara los datos de los request y de los response a medida que el usuario interactúe con la página. Esto es un grave problema teniendo en cuenta la naturaleza de un sistema web, ya que no tendría sentido estar preguntando al usuario cada vez que se llama una página, o que parámetros envió en los formularios.

    Imagina cómo trabaja un sitio real del calibre de Mercado Libre, por ejemplo; inicias sesión solo una vez y, a medida que vas navegando en el sitio, puedes ver que tu nombre de usuario se mantiene “vivo” entre página y página, permitiendo que agregues productos a tu carrito de compra. Esos productos están siendo añadidos bajo el alero de tu nombre de usuario (de tu sesión) y no se perderán, incluso si cambias de pestaña. Esto asegura la compra y la ﬁabilidad de los datos que se van almacenando.

    Este funcionamiento es posible gracias al manejo de sesiones, en una aplicación web y a las cookies.

    En la terminología web, una sesión es simplemente el intervalo de tiempo límite en que dos sistemas se comunican entre sí. Los dos sistemas pueden compartir información mediante arquitectura cliente servidor o arquitectura punto a punto. Sin embargo, con el protocolo http el estado de la comunicación no se mantiene. A causa de esto, las aplicaciones web tienen que utilizar una serie de técnicas que permitan mantener el estado entre cliente y servidor en el transcurso del tiempo. Cuando hablamos de "mantener el estado" nos referimos a guardar los datos del usuario y su información relacionada, para que independiente de la navegación del sitio, se siga reconociendo.

    Las cookies son la técnica más comúnmente usada para mantener los valores del usuario en sesión, pero si bien cumple con su función no estamos exentos de desventajas:

    • Las cookies sólo pueden mantener información textual.

    • Son dependientes del navegador, esto signiﬁca que si el cliente deshabilita las cookies, ya no será posible utilizarlas.

    • No puede almacenar más de 4kb de información.

    Para solventar estos problemas, los servlets implementan su propio mecanismo de manejo de sesiones. La imagen siguiente graﬁca como trabaja la sesión la tecnología java con servlets:

    La biblioteca de java nos provee de las clases HttpSession , la cual nos permite manejar y almacenar variables de sesión en torno a las acciones del usuario. Cuando hablamos de variables de sesión, estamos diciendo que esta clase puede almacenar cualquier tipo de elemento, incluidas las clases de la aplicación (entidades de negocio).

    EjemploSesion.java

    Esta clase, es la encargada de iniciar la sesión, es acá donde se inician algunos valores que luego serán consultados en otros contextos.

    package cl.lherrera.servlets;

import java.io.IOException;
import java.io.PrintWriter;
import java.util.Date;

import javax.servlet.annotation.WebServlet;
import javax.servlet.http.HttpServlet;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;
import javax.servlet.http.HttpSession;

import cl.lherrera.modelos.EntradaConcierto;

@WebServlet("/creaSesion")
public class EjemploSesion extends HttpServlet {

    /**
     * 
     */
    private static final long serialVersionUID = 1L;

    private String textoHTML(EntradaConcierto entrada) {
        StringBuilder retorno = new StringBuilder();

        retorno.append("<!DOCTYPE html>                                                                ");
        retorno.append("<html lang=\"en\">                                                             ");
        retorno.append("<head>                                                                         ");
        retorno.append("    <meta charset=\"UTF-8\">                                                   ");
        retorno.append("    <meta name=\"viewport\" content=\"width=device-width, initial-scale=1.0\"> ");
        retorno.append("    <title>Document</title>                                                    ");
        retorno.append("</head>                                                                        ");
        retorno.append("<body>                                                                         ");
        retorno.append("  <h1>Entradas para: " + entrada.getNombreArtista() + "</h1>                   ");
        retorno.append("  <h1>A nombre de: " + entrada.getTitularEntrada() + "</h1>                    ");
        retorno.append("</body>                                                                        ");
        retorno.append("</html>                                                                        ");

        return retorno.toString();

    }

    @Override
    protected void doGet(
            HttpServletRequest request, 
            HttpServletResponse response) throws IOException {
        String usuario = request.getParameter("nombre");
        EntradaConcierto entrada = new EntradaConcierto();

        // obtenemos la sesión del request.
        HttpSession sesion = request.getSession(true);

        // llenamos el objeto modelo con información de prueba
        entrada.setTitularEntrada(usuario);
        entrada.setNombreArtista("Adam Levine");
        entrada.setFechaConcierto(new Date());
        entrada.setUbicacion("desubicado");

        // agregamos atributos en la sesión, ("clave": valor)
        sesion.setAttribute("datosCompra", entrada);

        // seteamos información en el response
        PrintWriter writer = response.getWriter();
        writer.println(textoHTML(entrada));

    }

}

    Vanilla - JSP (scriptlet)

    Hay que demostrar que el uso de JSTL, es la forma en que se trabajará con JSP; pero en muchas ocaciones, nos encontraremos con desarrollos que no poseen implementación de estas librerías, es por ese motivo que debemos saber como utilizar java, en un archivo JSP.

    La forma de insertar código en un archivo JSP, es usando:

    <% 
    // todo el código de java acá ...
%>

    De esta forma, podemos en nuestros archivos jsp, insertar lógica, utilizando el lenguage Java, de forma "imperativa" + (um poco de composición). Es importante recordar que un JSP, es verdaderamente un Servlet en tiempo de ejecución. Es por esta razón que podemos incluir codigo Java junto a las etiquetas HTML, pero debemos indicar esta información, utilizando las etiquetas <%%>

    Para imprimir un mensaje por consola:

    <% System.out.println("Hola mundo"); %>

    Para imprimir HTML

    <%="Hola mundo" %>

    No debemos confundir un código que imprime por pantalla del que imprime en la página HTML, es decir que:

    <% System.out.println("Hola mundo"); %>  != <%="Hola mundo" %>
