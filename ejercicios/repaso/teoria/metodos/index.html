    Algunas reglas.

    • El tipo de retorno en un método jamás se omite (sólo el constructor no lo lleva).

    • Los nombres de los métodos siguen las mismas reglas que los nombres de las variables.

    • La excepciones que arroja el método pueden ser más de una, separadas con comas.

    • Solo se permite un varargs... por método y este debe ser el último en la lista de parámetros.

    • Si no ponemos el modificador de acceso (private, protected, public) en un método, este no queda público, lo que sucede realmente es que queda en una especie de privado para el paquete default, si importamos el paquete donde está este método sin el modificador, no podrá ser usado desde ese paquete externo. Solo podría comportarse como público dentro del paquete al que pertenece la clase que lo contiene.

    • Llamar a métodos estáticos de alguna instancia, puede provocar algunas confusiones como por ejemplo, la línea 8 seguirá imprimiendo lo que retorne count(), esto se debe a que para el compilador una vez que ya sabe que k.count() es por ejemplo 3, lo guarda como un valor, aunque k ahora apunte a null.

    • El constructor se sobrecarga como cualquier método.

    Constructor Chaining

    • Constructor Chaining o encadenamiento de constructores, es la técnica, en la que cuando hay una sobrescritura de un constructor, este puede llamar a otro constructor usando this.

    Sobre carga de métodos

    • Una clase puede tener varios métodos, con el mismo nombre. Esto es conocido como sobrecarga.

    • Dos varargs, producen ambigüedad, Integers... && int... nums. A diferencia de que si lo hacemos con un argumento simple Integer a o int a, si no es un varargs.. args no produce ambigüedad.

    • Existen dos maneras de sobrecargar un método.

      • Cambiando el número de argumentos.
      • Cambiando el tipo de dato de los argumentos.
      • No es posible hacerlo cambiando el tipo de retorno para evitar ambigüedades.

    Ejemplo:

    package cl.lherrera.java003;

public class Principal {

    public static void main(String[] args) {
        procesa();
        procesa(1);
        procesa(1, 2, 3);
        procesa("1");
    }

    private static void procesa() {
        System.out.println("Ejecutando: private static void procesa()");
    }

    private static void procesa(int a) {
        System.out.println("Ejecutando: private static void procesa(int a)");
    }

    private static void procesa(int... a) {
        System.out.println("Ejecutando: private static void procesa(int... a)");
    }

    private static void procesa(String a) {
        System.out.println("Ejecutando: private static void procesa(String a)");
    }

}

    Salida

Ejecutando: private static void procesa(int a)

Ejecutando: private static void procesa(String a)
